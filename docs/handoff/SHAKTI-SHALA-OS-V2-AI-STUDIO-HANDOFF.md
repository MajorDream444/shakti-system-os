# Key Corrections From Prior Design Pass

## 1. Do not expose prompt text in the interface

The previous pass treated editorial guidance as page copy.

Rule:

```text
Anything written in the prompt is instruction, not interface copy.
```

## 2. The primary world is Himalayan, not Bali

Bali may host retreats, but it is not the mythic center of the product.

The world should feel inspired by:

- Himalayas
- sacred mountain temples
- Kashmir Shaivism
- rivers
- snow light
- fire
- stone
- silence
- pilgrimage

Avoid tropical retreat aesthetics.

## 3. Access should feel like initiation, not subscription

Use:

```text
Visitor
Seeker
Practitioner
Initiate
Temple Circle
Guide
Teacher
```

Avoid:

```text
Basic
Pro
Premium
Upgrade
```

## 4. Tokenized access should feel like keys, not crypto

Public language:

```text
Temple Key
Initiation Key
Circle Access
Retreat Passage
```

Internal logic may include token gating and wallet checks, but seeker-facing design should stay human and ceremonial.

## 5. Google Drive is the living library

The app should reference Drive content beautifully instead of becoming a content dump.

Google Drive holds:

- PDFs
- videos
- audio
- worksheets
- retreat documents
- ceremony recordings
- source images

GitHub holds the app and design system.

Airtable tracks readiness and access.

Notion runs operations.
