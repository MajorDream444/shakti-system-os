## Sprint Classification Rule

Every Shakti sprint should label its work:

```text
Universal MAIM OS improvement
Shakti-specific implementation
Both
```

Examples:

| Work Item | Classification |
| --- | --- |
| BackendRepository abstraction | Universal MAIM OS |
| Shakti Temple Library language | Shakti-specific |
| Drive -> Airtable Library Assets approval path | Both |
| Mahavidya chamber visuals | Shakti-specific |
| Client-instance standard | Universal MAIM OS |
| Retreat readiness schema | Both |

## Guardrails

- Preserve Shakti visuals, sanctuary language, room architecture, and current
  interactions.
- Keep unapproved teachings out of the public sanctuary.
- Keep private Airtable tokens out of the browser in production.
- Keep Shakti-specific doctrine distinct from reusable MAIM OS standards.
- Promote reusable operating patterns upward only after they are clear enough
  to help future client instances.
