# Team Operations Playbook

## Operating Rhythm

- Keep GitHub focused on source code, durable docs, schema, and handoff notes.
- Keep Google Drive focused on source files and production assets.
- Keep Airtable focused on structured metadata and operational state.
- Keep Notion focused on team decisions and human-readable dashboards.

## Review Loop

1. Review new Drive assets.
2. Decide whether each asset stays local, moves to Drive, or enters GitHub.
3. Create Airtable metadata only for reviewed assets.
4. Update docs when workflows change.
5. Refresh Graphify after meaningful repo changes.
