# Doctrine Extraction Checklist

## Status

Sprint 11 editorial and review checklist.

Use this checklist when turning source material into doctrine notes and
downstream content.

## Purpose

Doctrine extraction identifies the teaching beneath the content.

It prevents the system from producing generic summaries, flattening sacred
language, or creating derivative experiences that drift away from the source.

## Source Identity

Record:

- source title
- speaker / teacher
- date
- source format
- Drive URL
- transcript URL or file path
- related offer / pathway / container
- confidentiality level
- intended audience

## Core Teaching

Extract:

- one-sentence thesis
- what problem the teaching addresses
- what the teaching invites
- what the teaching refuses or corrects
- primary metaphor or image
- key terms
- practice implications
- pathway implications

## Doctrine Questions

Answer:

1. What is the source really about beneath the surface topic?
2. What language must be preserved exactly or reviewed carefully?
3. What distinction does this teaching make?
4. What does this teaching protect against?
5. What kind of seeker is ready for this?
6. What kind of seeker may need more support before receiving it?
7. What practice, reflection, or next doorway belongs with it?
8. What should not be turned into marketing copy?

## Lineage And Language Review

Flag all:

- Sanskrit terms
- mantras
- deity names
- goddess / archetype correspondences
- philosophical frameworks
- ritual references
- ceremony references
- trauma / somatic / therapeutic claims
- retreat claims
- initiation or access language

Do not silently correct these. Mark them for Sheetal/team review.

## Output Notes

For each derivative output, identify:

| Output | Required Doctrine Anchor |
| --- | --- |
| Podcast Episode | teaching arc and listening doorway |
| E-book Chapter | chapter thesis and integration arc |
| Website Teaching | public-safe thesis and next doorway |
| Reflection Questions | approved inquiry frame |
| Social Carousel | one distinction, not the whole doctrine |
| Instagram Reel | one spoken teaching moment |
| Newsletter | relationship-forward teaching and reflection |

## Review Status

Use one of:

```text
Raw
Needs Review
Needs Transcript
Needs Summary
Ready To Tag
Approved
Published
Archived
```

Doctrine notes should not be treated as final while the source is below
`Approved`.

## Knowledge Maturity

Update `Knowledge Maturity` as the source moves through production:

```text
Raw Recording
Transcript Generated
Transcript Reviewed
Doctrine Extracted
Media Tagged
Library Ready
Sheetal Approved
Published
Repurposed Complete
```

For doctrine extraction work, the key maturity transition is:

```text
Transcript Reviewed -> Doctrine Extracted
```

## First Teaching Checklist

For `The Vow Beneath the Path`, confirm:

- Sankalpa is framed as vow, not wish or affirmation
- Sankalpa is described as gestated, not manufactured
- Atma Tattva Avalokanam is spelled and framed correctly
- Hiranyagarbha / Golden Womb language is approved
- Purusharthas are named and contextualized correctly
- goddess correspondences are approved
- Chinnamasta framing is not over-simplified
- Gupt Navratri language is approved
- 40-day devotional path is accurate
- public/private boundary is clear

## Final Gate

Doctrine extraction is complete only when:

- review flags are resolved or explicitly carried forward
- source status is updated
- approved language is documented
- derivative experiences cite the doctrine notes
- the Temple Library record reflects the final metadata

## Human Approval Gate Alignment

Doctrine extraction supports:

```text
Gate 2 — Doctrine Integrity
Did extraction preserve meaning?
```

It does not complete:

```text
Gate 3 — Lineage Integrity
Gate 4 — Publication Integrity
```

Those remain human approval gates before publication.
